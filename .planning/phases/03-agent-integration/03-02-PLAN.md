<task type="auto">
  <name>Task 1: End-to-end trough floor calculation for BRBR and PYPL</name>
  <files></files>
  <action>
For each of BRBR and PYPL, follow the analyst's 5-step structured process exactly as written in the updated analyst.md:

**BRBR (BellRing Brands):**

1. **Step A -- Identify trough inputs from FMP data:**
   ```bash
   bash scripts/fmp-api.sh income BRBR
   bash scripts/fmp-api.sh cashflow BRBR
   bash scripts/fmp-api.sh metrics BRBR
   bash scripts/fmp-api.sh profile BRBR
   ```
   From the 5yr history, identify:
   - Lowest trailing-12-month revenue
   - Lowest annual FCF margin (FCF / revenue for each year)
   - Industry baseline multiple from valuation-guidelines.md, minus applicable discount schedule
   - Current diluted shares outstanding

2. **Step B -- Run floor calculator:**
   ```bash
   bash scripts/calc-score.sh floor <brbr_rev_b> <brbr_margin> <brbr_multiple> <brbr_shares> <brbr_current_price>
   ```
   Record the full JSON output.

3. **Step C -- TBV cross-check:**
   ```bash
   bash scripts/fmp-api.sh balance BRBR
   ```
   Compute TBV/share and compare to floor_price.

4. **Step D -- Note any adjustment** (for this test: use mechanical floor, no adjustment).

5. **Step E -- Build trough path table** showing each input traced to a specific fiscal year.

6. **Reproducibility check:** Run the exact same floor command again and diff the output.

**PYPL (PayPal):**

Repeat the same 5 steps:
1. Pull FMP data (income, cashflow, metrics, profile, balance)
2. Identify trough inputs from 5yr history
3. Run calc-score.sh floor with trough inputs
4. TBV cross-check
5. Build trough path table
6. Reproducibility check

**Reasonableness check for both:**
- BRBR: Does the floor price land in a range that would have prevented the 12% vs 42% variance? (Mechanical floor should be deterministic regardless of who runs it)
- PYPL: Does the trough multiple from the discount schedule approximate the historical trough multiple (~8x) the analyst used?
- Neither floor should be absurdly negative without negative FCF margins in the data

**Output:** Write a brief verification report to `.planning/phases/03-agent-integration/03-e2e-verification.md` documenting:
- Trough inputs identified for each stock (with fiscal years)
- Floor calculator output (JSON)
- TBV cross-check result
- Trough path table
- Reproducibility confirmation
- Reasonableness assessment
