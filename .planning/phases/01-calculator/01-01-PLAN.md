Add a `floor` command to calc-score.sh that computes a mechanical worst-case floor price from trough inputs and returns the downside percentage from current price.

Purpose: Provides the deterministic anchor for downside estimation. Same trough inputs = same floor price = same downside percentage. Eliminates the LLM variance that produced 17-point score gaps (BRBR: 12% vs 42% downside) for the same stock on the same day.

Output: Working `floor` command in calc-score.sh with help text, input validation, and edge case handling.

Validation helpers (reuse directly):
```bash
validate_number() {
    local name="$1" val="$2"
    if ! python3 -c "float('$val')" 2>/dev/null; then
        err "$name must be a number, got: $val"
    fi
}

validate_pct() {
    local name="$1" val="$2"
    validate_number "$name" "$val"
    # warns on 0.30 vs 30 style decimals
}
```

Valuation command (the template for floor — same formula):
```bash
valuation)
    [[ $# -eq 4 ]] || err "valuation requires 4 args: ..."
    validate_number "revenue_billions" "$1"
    validate_number "fcf_margin_pct" "$2"
    validate_number "multiple" "$3"
    validate_number "shares_millions" "$4"

    python3 -c "
import json
revenue_b = float('$1')
fcf_margin_pct = float('$2')
multiple = float('$3')
shares_m = float('$4')

if shares_m <= 0:
    raise ValueError('shares_millions must be positive')

fcf_margin = fcf_margin_pct / 100
fcf_b = round(revenue_b * fcf_margin, 4)
target_price = round((fcf_b * multiple * 1000) / shares_m, 2)

print(json.dumps({
    'revenue_b': revenue_b,
    'fcf_margin_pct': fcf_margin_pct,
    'fcf_b': round(fcf_b, 4),
    'multiple': multiple,
    'shares_m': shares_m,
    'target_price': target_price
}, indent=2))
"
    ;;
```

JSON output pattern: all commands echo inputs back, add computed fields, use `json.dumps(result, indent=2)`.
Error pattern: `err()` prints to stderr, exits 1.
